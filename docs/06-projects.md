# 06. 工程管理

## .kln 工程文件

KnotLink 使用 `.kln` 格式保存工程。这是一个 zip 包，内含：

```
project.kln
├── project.json    # 工作区（节点+连线）元信息
├── main.py         # 生成的 Python 代码
└── apps/           # 自定义 funclist（可选）
```

## 基本操作

### 新建

`Ctrl+N` 或 📄 按钮。清空画布，开始新工程。

提示：如当前有未保存修改，会弹出确认框。

### 打开

`Ctrl+O` 或 📂 按钮。弹出文件选择对话框，选择 `.kln` 文件。

也可以直接将 `.kln` 文件拖到 exe 图标上打开。

### 保存

`Ctrl+S` 或 💾 按钮。

- 如果文件是从对话框打开的 → 直接覆盖原文件
- 如果文件是拖入的 → 第一次保存会弹出另存对话框
- 如果从未保存过 → 弹出另存对话框

### 另存为

💾另存 按钮。始终弹出保存对话框，选择新位置。

## 文件名显示

工具栏左侧显示当前文件名：

- `未命名工程.kln` — 还没保存过
- `t1.kln *` — 有未保存的修改（星号标记）
- `t1.kln` — 已保存

## 导入导出

- **导出**：等同于"另存为"，打包工作区+代码为 .kln
- **导入**：等同于"打开"，恢复工作区和代码
